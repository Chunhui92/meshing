# MIR 后续开发路线图

## 目标

在当前 zoo-only 第一阶段原型基础上，把 MIR 逐步升级为：

- 支持一般多材料 mixed zone
- 拓扑更稳定
- 体积误差更低
- 验证更自动化
- 在大规模数据下更可控

当前已经具备：

- `cell -> node` 体积分数重采样
- 共享点缓存
- zoo-only 重建主线
- zone 内逐材料顺序裁剪
- 局部体积分数校正
- `src/` 与 `test/` 分层
- VTK 分析和可视化脚本

## 开发顺序

建议继续按下面顺序推进：

1. 稳定当前 zoo 内核
2. 做强 `>2` 材料验证
3. 继续降低 non-manifold 与退化片段
4. 升级回归测试
5. 最后再做性能与大规模数据

不要在当前阶段先做并行/GPU，再回头补算法正确性。

## Phase 1: 稳定当前 zoo 内核

目标：

- 降低 `non_manifold_edges`
- 减少退化单元
- 减少局部拓扑伪影

建议工作：

- 把当前顺序裁剪从“可运行”继续提升到更明确的 zoo case 分类框架
- 单独整理 `corner / edge / helper` 三类点的职责和缓存策略
- 补充 face 内部交点、近零边、重复点合并的显式规则
- 减少当前 hull-based tet 化带来的局部碎片

验收：

- `boundary_edges` 保持 `0`
- `non_manifold_edges` 持续下降
- 体积误差不明显恶化

## Phase 2: 做强 `>2` 材料验证

目标：

- 不只是在代码路径上支持多材料，而是在专门数据上稳定验证

建议工作：

- 增加人工 3/4 材料测试集
- 覆盖薄层材料、小材料过滤、三材料交汇、接近退化界面
- 对比过滤前后材料保留策略是否稳定

验收：

- 3/4 材料样例可稳定输出
- VTK 可读取、可分析、可可视化
- 无 `plane` fallback 依赖

## Phase 3: 继续降低体积误差和拓扑问题

目标：

- 让局部校正更稳，减少 topology / volume 的相互牵制

建议工作：

- 改进当前一维二分校正的初始化与停止准则
- 统计每轮校正的局部误差和失败 zone 分布
- 针对失败 zone 增加更稳的局部处理，而不是全局改拓扑

验收：

- 总体积误差进一步下降
- `volume_correction_failed_zones` 保持低位
- 不明显增加 non-manifold 或退化单元

## Phase 4: 回归测试升级

目标：

- 把分析脚本升级为真正的回归门禁

建议工作：

- 固化 smoke/regression 数据集
- 为 `test/analyze_vtk.py` 补更多阈值项
- 固定当前可接受的体积误差和拓扑上限
- 在超阈值时稳定返回失败状态

验收：

- 能稳定复现当前样例结果
- 可作为后续改动的回归闸门

## Phase 5: 性能与大规模数据

目标：

- 在算法正确性稳定后，再处理时间和内存问题

建议工作：

- 评估 chunked writer
- 评估 edge cache / helper point 的内存增长
- 考虑两阶段输出：先计数，再分配，再写入
- 如果确实有必要，再考虑 CPU 并行、prefix-sum 输出或 GPU 路线

当前不建议优先做：

- 在 `zoo` 仍有明显 non-manifold 问题时直接做 GPU 化
- 在多材料专门测试集缺失时投入大量性能优化

## 建议测试集

- 当前样例：
  - `data/rect3d.silo`
  - `data/ucd3d.silo`
  - `data/multi_ucd3d.silo`
- 小规模 smoke case：
  - `--max-zones` 子集
- 人工构造 case：
  - 二材料平面切割
  - 薄层材料
  - 三材料交汇
  - 四材料交汇
  - 小材料被 `--min-fraction` 过滤
 - 材料分布检查：
  - `data/torus_mat.silo` 可用于确认 `>2` 材料 zone 的存在，但当前 mesh 类型尚不在 3D 重建支持范围内

## 当前最值得做的一步

优先补齐 **专门的 `>2` 材料 regression 数据集和 zoo case 回归**。

原因：

- 当前代码路径已经支持顺序多材料裁剪，但仓库内真实样例仍只有 1/2 材料 zone
- `multi_ucd3d.silo` 增加了 multiblock UCD 验证覆盖，但仍然没有 `>2` 材料 zone
- `torus_mat.silo` 证明仓库已出现三材料材料块数据，但还缺可直接用于当前 3D zoo 重建的对应 mesh 支持或等价 3D 数据
- 没有专门多材料样例，就无法可靠评估体积误差、拓扑稳定性和过滤策略是否真的成立
- 这一步会直接决定后续是继续补 case-table，还是先收敛几何裁剪与校正策略
